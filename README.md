# BitStoreA lightweight, high-performance Key-Value cache and message broker built entirely from scratch in C++.
## 🚀 OverviewBitStore is an ongoing systems programming project designed to explore how production databases like Redis work under the hood. It features a custom multi-threaded TCP server, active memory management, persistent storage, and real-time messaging capabilities.
## ✨ FeaturesIn-Memory Key/Value Store: Blazing fast $O(1)$ lookups using standard C++ hash maps.Time-To-Live (TTL) & Garbage Collection: Keys can be assigned an expiration time. A dedicated background thread safely sweeps and reclaims memory.Data Durability (AOF): Uses an Append-Only File log to persist data to disk in $O(1)$ time, with manual log compaction to prevent disk bloat.Task Queues: Native support for double-ended queues for producer/consumer workloads.Real-Time Pub/Sub: 1-to-many message broadcasting system with robust handling of dropped client connections (SIGPIPE armoring).Custom REPL Client: Includes a dedicated CLI tool (bitstore-cli) for an intuitive user experience.
